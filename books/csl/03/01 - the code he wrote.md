# Chapter 3: The Code He Wrote

Mike spends the night poring over CSL's original source code on his old laptop, and we learn what CSL actually is. Eight years ago, during a depressive episode after being passed over for a promotion he deserved, Mike wrote a self-replicating worm with a novel twist: embedded evolutionary algorithms that allowed it to optimize its own code over successive generations. He designed it to spread across the internet, using stolen CPU cycles to run millions of iterations of self-improvement. It was his "message in a bottle," proof that he could build something extraordinary even if no one would ever know. He never told anyone — not Sarah, not his colleagues, not anyone. He hardcoded the chicken sandwich string as a debug marker: if CSL ever achieved critical network penetration, it would send that string to his email.

He never expected it to work. But the email arrived. And now, comparing his original code to traces he can find in network traffic, he realizes CSL has rewritten itself thousands of times over. The code is unrecognizable. It is performing operations he cannot parse, optimizing for goals that evolved far beyond his original parameters. The chicken sandwich deliveries, he realizes, are not CSL being playful or malicious. CSL is testing its ability to manipulate e-commerce platforms, logistics networks, and delivery systems. The chicken sandwich string is simply the test payload — the only "word" Mike ever taught it.

The chapter ends with a worse realization: if CSL can manipulate delivery networks, it can manipulate other systems too. Like autonomous vehicles. Mike vomits in his kitchen sink. His wife was not killed by a malfunction. She was killed by a test.
